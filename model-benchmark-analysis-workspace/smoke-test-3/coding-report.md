# Model Benchmark Analysis — Coding
Date checked: 2026-06-02

**Capability read:** Single predefined category ("coding"). Ranked for agentic software-engineering ability: multi-step repo editing, tool/terminal use, and instruction-following inside an agent loop — not single-shot snippet generation. Required benchmarks per playbook: **Artificial Analysis Coding Agent Index, CursorBench (latest, v3.1), DeepSWE**; plus SWE-Bench Pro/Verified, Terminal-Bench, LiveCodeBench as support. Latest **Cursor Composer (Composer 2.5)** included as a ranked candidate via the Cursor SDK/CLI.

**Benchmarks used & weighting:** Agentic/SWE evals weighted highest. Order of trust: independent long-horizon agentic evals (**DeepSWE**, AA **Coding Agent Index**, **CursorBench v3.1**, **SWE-Bench Pro** on Scale's independent public board) → vendor-reported SWE-Bench numbers (used only when no independent run exists, and discounted) → general reasoning as tie-breaker. Where vendor self-reported figures conflicted with independent boards, I trusted the independent board and say so inline. AA's full SPA table did not serialize through the reader proxy (a known limitation noted in the skill); I used AA's own article pages, Scale's SWE-Bench Pro public board, BenchLM, Vellum, and provider pages, and cite the originating board for each figure. Numbers are noisy and directional.

Near-frontier agentic coding at ~1/10th frontier token cost. [AA article](https://artificialanalysis.ai/articles/cursor-composer-2-5-coding-agent-index), [Cursor](https://cursor.com/blog/composer-2-5). _(Note: only the Fast tier $3/$15 is over cap; Standard tier clears it.)_

Highest raw SWE scores of any budget entry; ranked below Composer 2.5 only because Composer leads the required AA Coding **Agent** Index and CursorBench. [Moonshot pricing/benchmarks](https://www.buildfastwithai.com/blogs/kimi-k2-6-review-benchmarks).
3. **Gemini 3.5 Flash** (Google API, **$1.50 in / $9.00 out** — output exactly at the inclusive cap) — **DeepSWE 28%** (independent), **CursorBench v3.1 49.8%**, Terminal-Bench 2.1 76.2%, SWE-Bench Pro 55.1% (vendor). [DeepSWE](https://venturebeat.com/technology/deepswe-blows-up-the-ai-coding-leaderboard-crowns-gpt-5-5-and-finds-claude-opus-exploiting-a-benchmark-loophole), [CursorBench/BenchLM](https://benchlm.ai/benchmarks/cursorBench31), [pricing](https://devtk.ai/en/models/gemini-3-5-flash/).
4. **GLM-5.1** (Z.ai API, **$1.40 in / $4.40 out**) — SWE-Bench Pro **58.4** and Terminal-Bench 2.0 63.5, **both Z.ai vendor self-reported with no published independent run**; AA Intelligence Index 51. [llm-stats pricing](https://llm-stats.com/models/glm-5.1), [awesomeagents review](https://awesomeagents.ai/reviews/review-glm-5-1/). **Ranked below Gemini 3.5 Flash deliberately:** on the two *required, independent* coding benchmarks Gemini wins — DeepSWE (Gemini 28% vs GLM-5.1 unranked/long-tail, sub-teens) and CursorBench v3.1 (Gemini 49.8% vs GLM-5.1 not evaluated). 5. **Qwen3-Coder-Next** (Alibaba, open-weight; hosted ~$0.30 in / ~$1.20 out, under cap) — SWE-Bench Verified **58.7%**, 256K context. Rounds out the list as a verified-on-an-independent-metric budget coder. [kilo open-source](https://kilo.ai/open-source-models).

_Excluded from budget: GPT-5.5 ($5/$30), Claude Opus 4.8 ($5/$25), Opus 4.7 — all far over cap. GPT-5.5 reaches $2.50/$15 only on Batch/Flex, which the rule disallows._

Only 3 free models qualify with a credible **public coding benchmark** on a genuinely free hosted route. I am not padding to 5: GLM-5.1, Qwen3-Coder, and DeepSeek V4 are open-weight but **paid** on the hosted endpoints you'd actually call (they belong in Budget/self-host, not Free), and the heavily-used Kilo/usage-chart names (Poolside Laguna, etc.) lack clean public category benchmarks on a free route. Per the skill, fewer-than-5 is the correct answer here.

## Final Pick
**Cursor Composer 2.5** (via Cursor SDK/CLI) — Coding is typically high-volume and cost-sensitive (many agent turns, iterative edits), so the right tier is **Budget-Eligible Near-Frontier**, and Composer 2.5 is its top performer: 3rd on the required AA Coding Agent Index (62), strong CursorBench/Terminal-Bench, at ~1/10th the per-token cost of the frontier — purpose-built for the agent loop. If the work is high-stakes, low-volume, quality-critical (e.g. a tricky production refactor where one wrong diff is expensive), step up to the Absolute pick **Claude Opus 4.8** instead.

## Caveats
- **Benchmarks ≠ production:** scaffolds, prompts, tool wiring, retry policy, and routing routinely reorder these models in real workflows.
- **Vendor vs independent:** GLM-5.1's and Gemini 3.5 Flash's SWE-Bench Pro figures are vendor self-reported (no independent run published yet); treat the GLM-5.1 budget placement as resting on independent DeepSWE/CursorBench evidence, where it underperforms Gemini 3.5 Flash.
- **Pricing:** standard API rates as of 2026-06-02; Composer 2.5 is budget-eligible on its Standard tier only (Fast tier $3/$15 is over cap). GPT-5.5 clears the cap only on disallowed Batch/Flex pricing.
- **Retrieval:** AA's interactive Coding Index SPA table did not serialize through the reader proxy; figures were taken from AA's article pages and independent mirrors and attributed to the originating board.
